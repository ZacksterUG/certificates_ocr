# OCR Service

Сервис для распознавания текста из изображений и PDF с использованием PaddleOCR-VL.

## Архитектура

1. Получает задачи из RabbitMQ очереди `text_recognition`
2. Загружает изображение/PDF из PostgreSQL (таблица `certificates`)
3. Обрабатывает моделью PaddleOCR-VL
4. Обновляет статус задачи в БД (`status = 'text_recognition'`, `raw_text = результат`)

## Запуск

### 1. Поднять инфраструктуру (RabbitMQ + PostgreSQL)

```bash
cd dev
docker-compose up -d
```

### 2. Установить зависимости

```bash
cd dev/ocr_service
pip install -r requirements.txt
```

### 3. Запустить сервис

```bash
python main.py
```

## Тестирование

### Через Jupyter Notebook

1. Запусти `main.py` в фоне
2. Открой `test.ipynb`
3. Выполняй ячейки по порядку:
   - **Ячейка 1**: импорты и настройки
   - **Ячейка 2**: загрузка `test_1.jpg` в БД со статусом `pending`
   - **Ячейка 3**: отправка задачи в RabbitMQ
   - **Ячейка 4**: проверка статуса (после обработки)

### Вручную через SQL

```sql
-- Проверка статуса
SELECT id, status, raw_text, error_message, completed_at 
FROM certificates 
ORDER BY created_at DESC 
LIMIT 1;
```

## Формат сообщения в очереди

Сообщение в RabbitMQ должно содержать JSON:

```json
{
  "id": "uuid-задачи"
}
```

## Логирование

Сервис выводит логи в консоль:
- `🔌` — подключение к сервисам
- `📥` — получение задачи
- `📷` / `📄` — обработка изображения/PDF
- `✅` — успешное обновление
- `❌` — ошибка

## Примечания

- Сервис обрабатывает **одну задачу за раз** (не масштабируемая версия)
- Для PDF используется первая страница
- При ошибке обработки статус устанавливается в `error`
